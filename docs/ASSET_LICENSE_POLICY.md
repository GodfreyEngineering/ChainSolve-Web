# Asset License Policy

This document defines the rules for including third-party assets
(fonts, icons, images, sounds, data files) in the ChainSolve codebase.

## Rules

1. **No unlicensed assets.** Every third-party asset checked into the
   repository or loaded at runtime must have a known, documented license.

2. **Permissive licenses only.** Acceptable licenses include: MIT, ISC,
   BSD-2-Clause, BSD-3-Clause, Apache-2.0, OFL-1.1, CC0-1.0, CC-BY-4.0,
   Unlicense, and 0BSD. Dual-licensed assets under any of these are also
   acceptable.

3. **Forbidden licenses.** The following license families are not
   permitted for inclusion in a commercial product:
   - GPL-2.0, GPL-3.0 (copyleft)
   - AGPL-1.0, AGPL-3.0 (network copyleft)
   - SSPL (Server Side Public License)
   - CC-BY-SA (share-alike)
   - CC-BY-NC (non-commercial)

4. **Attribution.** All third-party assets must be listed in
   `THIRD_PARTY_NOTICES.md` with their name, license type, and source.
   This file is auto-generated by `scripts/generate-licenses.mjs`.

5. **Fonts.** ChainSolve uses Google Fonts loaded from the CDN:
   - Montserrat (OFL-1.1)
   - JetBrains Mono (OFL-1.1)

   No font files are stored in the repository. If a local font is added
   in the future, its license file must be committed alongside it.

6. **Brand assets.** Files in `public/brand/` are proprietary ChainSolve
   assets. They are not third-party and require no external license.

7. **Icons and illustrations.** If third-party icons or illustrations are
   added, their license must be verified before merging and they must be
   added to the asset inventory in `scripts/generate-licenses.mjs`.

## Enforcement

The CI pipeline runs `node scripts/generate-licenses.mjs --check` to
verify that `THIRD_PARTY_NOTICES.md` is up to date and that no forbidden
licenses are present. This check must pass before any commit is merged.

## Updating the inventory

When adding or upgrading a dependency (npm or Cargo):

```sh
node scripts/generate-licenses.mjs
```

Review the diff to `THIRD_PARTY_NOTICES.md` and commit the updated file.
If the script reports a forbidden license, the dependency must not be used.
